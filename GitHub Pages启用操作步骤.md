# 🎯 GitHub Pages 启用操作步骤（图文版）

## ✅ 你现在在正确的页面！

页面显示：
- "Build and deployment"（构建和部署）
- "Source"（源）
- "Branch"（分支）下拉菜单

---

## 📝 操作步骤

### 第一步：选择分支

1. **找到 "Branch"（分支）部分**
   - 你会看到一个下拉菜单，当前显示 "None"

2. **点击下拉菜单**
   - 点击显示 "None" 的下拉菜单

3. **选择分支**
   - 在下拉菜单中选择 **"main"**
   - （如果没有"main"，选择"master"）

### 第二步：选择文件夹（如果有）

选择分支后，可能会显示 "Folder"（文件夹）选项：
- 选择 **"/ (root)"**（根目录）
- 这是默认选项，通常不需要更改

### 第三步：保存设置

1. **点击 "Save"（保存）按钮**
   - 选择分支后，"Save"按钮会变成可点击状态（不再是灰色）

2. **等待保存完成**
   - 页面会刷新
   - 显示保存成功的提示

### 第四步：等待部署

1. **等待1-2分钟**
   - GitHub会自动部署你的网站
   - 页面会显示部署状态

2. **查看部署结果**
   - 部署完成后，页面顶部会显示：
     ```
     Your site is live at:
     https://zy593.github.io/xiangji/
     ```

## 🎯 完整操作流程

```
1. 在 "Branch" 下拉菜单中选择 "main"

   ↓

   ↓

```

## ⚠️ 注意事项

1. **如果没有"main"分支**
   - 选择 "master" 分支
   - 或者检查仓库是否有代码

2. **如果Save按钮还是灰色**
   - 确保已选择分支
   - 刷新页面重试

3. **如果部署失败**
   - 检查仓库中是否有 `index.html` 文件
   - 确保文件在根目录

## ❓ 常见问题

**Q: 选择分支后没有反应？**
A: 点击 "Save" 按钮保存设置

**Q: 保存后没有显示网址？**
A: 等待1-2分钟，刷新页面查看

**Q: 显示404错误？**
A: 确保仓库中有 `index.html` 文件，等待几分钟后重试
